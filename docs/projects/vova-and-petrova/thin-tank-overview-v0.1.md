# Thin-tank Overview — Vova & Petrova  
Версия 0.1

---

## 1. Что такое thin-tank

**Thin-tank** — это раздел сайта и логический модуль проекта «Вова и Петрова», выполняющий роль:

- маленького мозга проекта V&P,
- витрины механизма (архитектур, пайплайнов, автоматизаций),
- витрины процесса (машинных логов, архитектурных решений, историй разработки).

Если extended-mind — это большой мозг экосистемы utemix-lab,  
то thin-tank — локальный мозг для одного проекта, встроенный в общую систему.

---

## 2. Задачи thin-tank

1. **Показать, как устроен проект внутри**  
   - архитектура контентной машины,
   - работа с источниками (LLM, парсеры, репозитории),
   - связи между SMM-каналами и Базой Знаний.

2. **Показать, как проект создаётся**  
   - машинные отчёты о проделанной работе,
   - выдержки из machine-log,
   - эволюцию решений (ADR) применительно к V&P.

3. **Сделать инженерную часть понятной для продвинутых зрителей**  
   - авторские разборы,
   - визуальные схемы,
   - примеры пайплайнов.

4. **Связать V&P с extended-mind**  
   - тонкие ниточки между локальной архитектурой и общим “мозгом”,
   - демонстрация того, как extended-mind влияет на решения V&P.

## 3. Структура thin-tank (v0.1)

Предлагаемая структура раздела:

```text
thin-tank/
    overview/           — базовое описание (этот документ + краткие версии для сайта)
    architecture/       — схемы, пайплайны, описания модулей
    process-log/        — машинные отчёты (machine-log-VP condensed)
    adr/                — решения, специфичные для V&P
    knowledge-routes/   — как контент превращается в знание
    tools-and-stacks/   — стеки технологий, софт, сервисы


На сайте это может быть:

одна страница с блоками,

или несколько подразделов для удобства навигации.

4. Поток: контент → thin-tank → База Знаний

Персонажи и автор создают контент:

видео, стримы, посты, обзоры, эксперименты.

Этот контент попадает:

на YouTube, в Telegram, VK, Instagram,

и параллельно — в систему учёта (репозитории, таблицы, структуры данных).

Thin-tank:

фиксирует путь: кто создал → какой формат → по какой теме → для какого уровня аудитории,

связывает контент с сущностями Базы Знаний (инструмент, теория, кейс, страна, центр компетенции),

определяет, какие фрагменты идут в:

понятные карточки для обычных пользователей,

техническую документацию,

extended-mind (как сырьё для RAG).

В результате:

всё, что “пролетело через экран”,

перестаёт быть просто медиа,

становится структурированным знанием.

5. Thin-tank и story-layer

Thin-tank работает на стыке:

machine-log-VP — инженерная хроника:

какие модули разрабатывались,

какие связки API пробовали,

какие пайплайны сработали.

author-line (в контексте V&P):

рефлексии автора о том, почему решения принимались именно так,

сомнения, переборы вариантов, тупиковые ветки.

Thin-tank:

не прячет хаос,

но превращает его в читабельные слои:

“как это было на самом деле” (machine-log),

“что мы из этого вынесли” (обзор/конспект),

“что теперь является архитектурой” (ADR и схемы).

6. Связь с extended-mind

Thin-tank:

получает правила игры и решения от extended-mind (ADR, паттерны, онтологию),

возвращает опыт и кейсы:

что получилось,

что нет,

какие архитектуры оказались жизнеспособными.

Так V&P становится:

не только шоу-проектом,

но и тренировочным полигоном для всей системы мышления utemix-lab.

7. Для кого thin-tank

Для продвинутых зрителей

тех, кому интересно “как оно устроено внутри”.

Для инженеров и исследователей

кто захочет перенять отдельные механизмы,

посмотреть на архитектуры,

увидеть реальные пайплайны.

Для самого автора

как инструмент самообразования,

как дневник инженерных решений,

как часть портфолио.

8. Статус документа

раздел будет расширен примерами, диаграммами и ссылками на реальные репозитории и страницы сайта.
