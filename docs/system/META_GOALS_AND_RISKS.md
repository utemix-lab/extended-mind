# META GOALS AND RISKS

Мета-цели — цели над целями. Они не про фичи, а про то, чем система имеет право быть.

## MG‑A. Навигация вместо онтологии
**Формулировка:** система помогает ориентироваться в сложных областях знаний и практик, не пытаясь описать мир целиком.  
**Риск:** онтологический дрейф.  
**Анти‑паттерн:** “У нас нет X — давайте добавим ещё один узел”.  
**Контр‑механизмы:** Canon минимален; Catalog ≠ Node; Query Mode вместо списков; Visual Guardrails.

## MG‑B. Контент‑фабрика без фабричности
**Формулировка:** система превращает свою работу и сигналы мира в контент, не становясь медиаплатформой.  
**Риск:** медийный захват.  
**Анти‑паттерн:** “Нужно хранить все видео, посты, реакции”.  
**Контр‑механизмы:** Story‑nodes как атом истории; контент = производная; Story как сцена; внешние платформы как проекции.

## MG‑C. Портфолио без каталога
**Формулировка:** демонстрация возможностей через следы работ и маршруты, а не списки объектов.  
**Риск:** каталогизация уникального.  
**Анти‑паттерн:** “Добавим карточку проекта со всеми полями”.  
**Контр‑механизмы:** Workbench как процесс; Saved Paths вместо “избранного”; Terminal Visual; нарратив вместо спецификации.

## MG‑D. Лаборатория, не замороженный продукт
**Формулировка:** система как место разработки самой себя и производных продуктов без переписывания архитектуры.  
**Риск:** архитектурная хрупкость.  
**Анти‑паттерн:** “Для этого нужен отдельный раздел / тип / сущность”.  
**Контр‑механизмы:** режимы ≠ сущности; practices как способы движения; steps/ADR/story‑nodes как память; contracts как язык.

## MG‑E. Самоописание без субъектности
**Формулировка:** система объясняет себя через данные и следы, не притворяясь “разумной”.  
**Риск:** псевдосознание.  
**Анти‑паттерн:** “LLM знает, как всё работает”.  
**Контр‑механизмы:** два слоя векторизации (архитектура/нарратив); story‑nodes как факты; SYSTEM_CHECKS и Steps как ground truth.

## MG‑F. Эмерджентность
**Формулировка:** система может стать чем‑то ещё, не описанным заранее.  
**Риск:** сверх‑жёсткость.  
**Анти‑паттерн:** заранее определить всё.  
**Контр‑механизмы:** минимальный канон; проекции вместо сущностей; слабая связность; Story как поле эксперимента.

## Сводная таблица рисков

| Риск | Где возникает | Чем лечится |
|---|---|---|
| Онтологический дрейф | Навигация | Catalog ≠ Node |
| Медийный захват | Контент | Story‑nodes |
| Каталогизация | Портфолио | Workbenches |
| Хрупкость | Лаборатория | Режимы |
| Псевдосознание | RAG/LLM | Два слоя |
| Сверх‑жёсткость | Рост | Эмерджентность |

## Character / Workbench (привязка)

**Character** — носитель фокуса и практик, точка сборки контента, публичная проекция системы.  
Связь: MG‑B, MG‑C, MG‑D, MG‑F.

**Workbench** — локальная архитектура действия, способ работы, контекст возможностей.  
Связь: MG‑A, MG‑C, MG‑D.

**Важно:** Workbench не узел канона — иначе он станет онтологией; он должен оставаться процессом.
