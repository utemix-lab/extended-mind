# SYSTEM INTENT

## Mission (one paragraph)
The system exists to keep a minimal, stable canon while enabling powerful navigation across large external catalogs. It orients users in complex domains without pretending to be a complete encyclopedia, and it keeps the architecture ready for future RAG/LLM enrichment without rewriting the core model.

## Promises to the user (3–5)
- ясная навигация по возможностям, а не “всё обо всём”
- минимальная когнитивная нагрузка: контекст важнее объёма
- прозрачные маршруты и объяснимые результаты
- устойчивость: изменения не ломают основу

## Promises to the lab/author (3–5)
- канон остаётся малым и управляемым
- данные можно масштабировать без роста онтологии
- решения фиксируются и остаются воспроизводимыми
- система готова к RAG без архитектурного разлома

## What the system does NOT do
- не строит “второй интернет”
- не превращает теги/страны/функции в узлы графа
- не хранит длинные тексты в каноне
- не навязывает единственный маршрут пользователю

## Mini‑manifest
Architecture is law.  
Narrative is life within the law.  
Law does not justify itself; life does not have to be consistent.
