# EMERGENT MODES (Practices & Opportunities)

## Определения

- **Node** = место (сущность графа)
- **Tag** = якорь запроса
- **Query Mode** = режим среза (state)
- **Practice** = способ мышления/движения (не сущность)
- **Opportunity** = действие/производство артефакта (не сущность)

## Принцип появления

Несущности проявляются как следствие контекста и не вызываются напрямую.

## Триггеры появления (минимум)

- Node selected
- Tag clicked (Query Mode active)
- Service clicked (refine query)
- Composite tag selected
- (Future) Opportunity invoked

## Правило количества

- Practices shown: max 3 (идеал 1)
- Opportunities shown: max 3 (идеал 1)
- Никаких “show all”

## Распределение по окнам (репетиционное)

- **Story**: interpretive hints (“мы в практике …”)
- **Service**: actions / query / results
- **System**: optional deepening (не dev‑панель)

## Non-goals / запреты

- Не превращать Practices/Opportunities в узлы
- Не делать отдельную страницу/хаб “All practices”
- Не делать “магическую кнопку показать всё”
- Не фиксировать жёстко Story/System/Service как таблицу сущностей
