## Контекст

В домене AI появились большие каталоги и необходимость ориентации без расширения канона.
Требовался механизм запроса, не превращающий теги в сущности и не создающий новые страницы.

## Решение

Ввести Query Mode как глобальный режим восприятия, запускаемый кликом по tag‑pill.
Теги остаются якорями запроса, каталоги — вне канона. UI‑реализация живёт в Service‑окне, но архитектурно режим не привязан к окну.

## Последствия

**Положительные:**
- Канон остаётся минимальным, каталоги — внешним слоем.
- Тег = запрос, а не навигация или сущность.
- UI остаётся гибким, Query Mode может быть переиспользован.

**Отрицательные:**
- Нужна дисциплина в каталогах и tag‑нормализации.
- Появляется отдельный слой поведения, который нельзя фиксировать в онтологии.

## Альтернативы

- Страницы под каждый сервис/модель — отклонено (раздувает UI и смешивает канон с каталогами).
- Capability‑узлы в каноне — отклонено (ломает принцип “каталог ≠ канон”).
