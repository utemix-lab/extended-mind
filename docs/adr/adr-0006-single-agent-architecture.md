# ADR-0006  
**Single-Agent Architecture for Extended-mind**

---

## 1. Контекст

Extended-mind — это проект автора-одиночки.  
У него нет команды DevOps / MLOps.  
Он работает в камерном режиме и не требует высокой распределённости.

Многоагентная архитектура выглядит:
- заманчиво,
- эффектно,
- концептуально красиво,

но усложняет систему раньше времени.

---

## 2. Решение

**Extended-mind v0.x–v1.0 остаётся одноагентной системой.**

- один голос,
- одна логика,
- одно метамышление,
- единая точка принятия решений.

Extended-mind может координировать IDE-агентов (Cursor, Copilot), но сам остаётся единым.

---

## 3. Обоснование

1. Простота = надёжность.  
2. Единая логика мышления формирует чистую онтологию.  
3. Многоагентность на старте приводит к рассогласованию.  
4. История создания проектов должна быть прозрачной.  
5. RAG на раннем этапе не даёт весомых преимуществ для распределённого поведения.

---

## 4. Последствия

Плюсы:
- высокий контроль качества,
- отсутствие “расползания” мыслей по агентам,
- легко поддерживать,
- легко объяснять публике.

Минусы:
- меньше параллелизма,
- часть решений придётся делать вручную.

---

## 5. Будущее

В будущих версиях (>1.0) можно рассмотреть:
- временных агентов-специалистов,
- агентные пайплайны,
- автоматизацию сценариев.

Но только после стабилизации онтологии.

---

## 6. Статус

**Принято.**
