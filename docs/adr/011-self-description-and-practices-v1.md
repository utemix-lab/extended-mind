## Контекст

Система получила два слоя для будущего RAG (архитектурный и нарративный), но не имела цели
самоописания и механизма обязательного нарративного следа для архитектурных изменений.
Одновременно расширяется набор практик, который задаёт осевые маршруты мышления и действия.

## Решение

1) Ввести цель самоописательности системы (LG/MG) и закрепить narrative trace для архитектурных сдвигов.  
2) Расширить список осевых практик v1 до 14 пунктов в `PRACTICES_SPEC.md`.

## Последствия

**Положительные:**
- каждое архитектурное решение получает narrative‑след
- система может описать себя для разных аудиторий
- практики становятся согласованной картой маршрутов

**Отрицательные:**
- требуется дисциплина фиксации story‑nodes
- добавляется слой документов для целей и сверки

## Альтернативы

- Оставить практики как неформальный список → риск расхождения и дрейфа.
- Ввести самоописание только в нарративе → потеря системной цели и контроля.
