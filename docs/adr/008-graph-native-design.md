## Контекст

В системе проявились три разных вещи, которые легко смешать:
1. Архитектурный подход (как система устроена)
2. Визуальный язык интерфейса (как система выглядит)
3. Художественные практики (что система может создавать)

## Решение

### Три независимых слоя

| Слой | Что это | Владелец |
|------|---------|----------|
| **Архитектура** | Метод проектирования | Система |
| **Визуальный язык** | Сценография интерфейса | Рендер |
| **Художественные практики** | Стиль + инструмент | Персонаж |

### Рабочее имя архитектурного подхода

**Graph-native system design** — графо-ориентированное проектирование систем.

Это не бренд, не манифест. Это рабочее имя класса.

## Ключевая формула

> Архитектура системы — graph-native и perception-separated.  
> Визуальные стили — практики внутри системы, не её архитектура.
