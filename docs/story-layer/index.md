# Story-layer Index  
Версия 0.1

Story-layer — сырьевой слой мышления.  
Он фиксирует всё, что предшествует структуре.

---

# 1. Состав story-layer

Story-layer состоит из трёх типов материалов:

1. **Author-line** — субъективные, художественные, интуитивные записи.  
2. **Story-node** — сырые фрагменты мысли: наблюдения, озарения, противоречия.  
3. **Machine-log** — инженерная хроника принятия решений.

---

# 2. Назначение каждого типа

### Author-line  
- художественный поток мыслей  
- литература о процессе  
- эмоциональные и философские фрагменты  
- материал для рилсов, рефлексивных текстов, образов

### Story-node  
- основное поле “мыслительной пены”  
- содержит зерна смысла  
- служит материалом для извлечения тезисов

### Machine-log  
- фиксирует ход работы extended-mind  
- содержит:  
  - вопросы  
  - диалоги  
  - промежуточные решения  
  - аргументацию  
- основа для документирования архитектуры

---

# 3. Как extended-mind работает со story-layer

### 3.1. Принципы

- Ничего не удалять → только помечать статусом.  
- Не смешивать типы записей.  
- Использовать story-layer только как сырьё, а не как структуру.  
- Применять паттерны преобразования.

### 3.2. Поток обработки

1. Автор создаёт story-node или author-line.  
2. Extended-mind извлекает смысл → тезисы.  
3. Тезисы объединяются → линии.  
4. Линии → карты смыслов.  
5. Карты → ADR.  

Story-layer **никогда напрямую не становится архитектурой**.

---

# 4. Статусы story-node

- `raw` — сырое, только что создано  
- `refined` — из него извлечены тезисы  
- `deprecated` — идея устарела, но хранится  
- `converted` — использовано в линии или ADR

# 6. Для чего нужен story-layer

Он позволяет:

- видеть эволюцию мысли,  
- понимать, как рождались решения,  
- защищать авторскую живую часть процесса,  
- обеспечивать возможность художественных переосмыслений.

Extended-mind относится к story-layer как к ресурсной базе.
