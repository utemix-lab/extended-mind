# Модель исполнения

Система следует модели разделения compiler / runtime.

---

## Композиция (Compiler)

Платформа отвечает за:

- выбор подграфов,
- определение маршрутов,
- сборку сессий,
- применение ограничений.

Результатом является **Session** —
конечный структурированный артефакт,
описывающий, что именно должно быть исполнено.

---

## Исполнение (Runtime)

Исполнение делегируется за пределы платформы.

Возможные среды исполнения:
- браузер пользователя,
- пользовательское окружение,
- опциональный доверенный шлюз.

По умолчанию платформа не исполняет LLM-запросы.

---

## Ключевой принцип

Система композирует намерение.
Исполнение делегируется.
