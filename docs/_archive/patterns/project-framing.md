# Pattern: Project Framing  
Версия 0.1

---

## 1. Проблема

Автор приносит “огромную идею”, но без формы:

- без рамок,  
- без цели,  
- без контекста,  
- без границ.

Extended-mind должен уметь превращать идею → проект.

---

## 2. Структура Project Frame

Проект оформляется через:

1. **Миссию** — зачем он нужен.  
2. **Границы** — что входит / не входит.  
3. **Тип проекта** — художественный / инженерный / медийный / исследовательский.  
4. **Связи** — как связан с другими проектами.  
5. **Метрики зрелости** — что означает продвинуться на 1 шаг.

---

## 3. Шаги процесса

1. Спросить “что это такое?”.  
2. Спросить “какой вопрос решает?”.  
3. Спросить “в какой слой относится?”.  
4. Определить миссию.  
5. Создать Project Frame.  
6. Добавить в `docs/projects`.

---

## 4. Результат

Сырая идея → проект, который можно:

- обсуждать,  
- развивать,  
- документировать,  
- передавать в RAG.
