# ADR-0003  
**RAG First, Graph Later**

---

## 1. Контекст

Extended-mind предполагает:
- онтологию,
- связи,
- слои,
- многопроходное мышление.

Графовые технологии (GraphRAG, knowledge graphs) выглядят естественным выбором.  
Однако их сложность, стоимость и необходимость строгих данных делают их неподходящими на раннем этапе.

---

## 2. Решение

1. **Extended-mind начинается с простого RAG.**  
   Простой векторный поиск:
   - прозрачен,
   - доступен,
   - требует минимальной инженерии.

2. **Граф будет добавлен позднее, когда появится стабильная онтология.**

3. **Оптимизация:**  
   Формировать документы так, чтобы в будущем они могли быть импортированы в граф.

4. **Граф = надстройка над RAG, а не замена.**

---

## 3. Обоснование

- RAG — быстрый и надёжный.
- Граф — требует стабильности, которой сейчас нет.
- Extended-mind должен быть камерным, доступным автору без DevOps-ресурсов.
- Векторные базы проще в обслуживании.

---

## 4. Последствия

Положительные:
- малый порог входа,
- низкая стоимость,
- быстрые итерации,
- ранний результат.

Ограничения:
- глубокие смысловые связи будут добавляться позже.

---

## 5. Статус

**Принято.**  
Графы появятся только в версии extended-mind v1.0+.
