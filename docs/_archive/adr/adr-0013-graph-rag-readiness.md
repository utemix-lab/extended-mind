# ADR-0013 — Criteria for GraphRAG Readiness

## Context

Extended-mind v0.1 использует классический RAG
(векторный retrieval + строгая культура документов).

GraphRAG и графовые БД рассматриваются как возможное развитие,
но их преждевременное внедрение несёт риск:

- избыточной сложности,
- роста когнитивной нагрузки,
- накопления мусорных связей.

Необходимо зафиксировать **чёткие критерии**, при которых переход к GraphRAG
становится оправданным, а не модным.

---

## Decision

Extended-mind **не использует GraphRAG по умолчанию**.

Переход к GraphRAG допускается **только при выполнении всех следующих условий**:

### 1. Онтология устоялась
- object-types стабилизированы;
- основные project-frames зафиксированы;
- роли сущностей больше не “плавают”.

### 2. Связи стали первичнее текста
- большинство вопросов пользователя формулируются как:
  - “как это связано с…”
  - “через что это связано…”
  - “какие сущности участвуют…”
- векторный поиск находит правильные куски,
  но **не даёт нужной навигации по отношениям**.

### 3. Появился повторяющийся паттерн связей
- связи между сущностями не разовые,
- а устойчивые (тип → тип → тип),
- их можно формализовать как граф.

### 4. RAG достиг предела полезности
- retrieval часто возвращает:
  - правильные фрагменты,
  - но без объяснения структуры системы;
- требуется не “ответ”, а “карта”.

### 5. Есть ресурс на поддержку
- автор понимает:
  - что такое граф,
  - какие типы узлов и рёбер нужны;
- есть готовность поддерживать граф как живую структуру.

---

## Consequences

### Положительные
- сохраняется простота extended-mind v0.1;
- RAG используется максимально эффективно;
- граф появляется только тогда, когда он действительно нужен.

### Ограничения
- сложные навигационные запросы временно решаются:
  - через concept-maps,
  - через project-frames,
  - через архитектурные пояснения.

---

## Notes

До выполнения критериев GraphRAG рассматривается как:

- исследовательский эксперимент,
- отдельный sandbox,
- но **не как часть ядра extended-mind**.

Любое решение о внедрении GraphRAG фиксируется новым ADR.
