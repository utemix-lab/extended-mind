# ADR-0010  
**Why Projects Must Be Modular**

---

## 1. Контекст

Экосистема utemix-lab содержит:

- extended-mind,  
- Vova & Petrova,  
- evoquant,  
- word-machine,  
- будущие проекты.

Каждый проект:
- имеет собственную логику,
- собственный темп,
- собственный уровень зрелости.

Если они будут слиты в одну структуру → хаос.

---

## 2. Решение

**Каждый проект — отдельный модуль.**

С модульностью:

- у каждого проекта свой репозиторий,  
- своя архитектура,  
- свои правила,  
- свой темп развития,  
- объединение только через extended-mind.

---

## 3. Обоснование

1. **Чистота.**  
   Нет смешивания философии evoquant с продакшеном V&P.

2. **RAG-эффективность.**  
   Локальные базы знаний легче поддерживать.

3. **Гибкость.**  
   Проекты могут развиваться независимо.

4. **Публичный слой проще.**  
   Каждому проекту — своя витрина; extended-mind — “кухня”.

5. **Инженерная устойчивость.**  
   Модули проще тестировать, обновлять, документировать.

---

## 4. Последствия

Плюсы:
- нет путаницы между слоями и проектами,
- extended-mind становится центральным “каркасом”,
- каждый проект может развиваться на своём уровне сложности.

Минусы:
- введение модульности требует дисциплины ссылок.

---

## 5. Статус

**Принято.**
