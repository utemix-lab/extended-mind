# ADR-0008  
**Role of Definition Pack**

---

## 1. Контекст

Extended-mind — это система мышления.  
Чтобы система работала, ей нужна онтология — набор чётких определений:

- слоёв,  
- понятий,  
- сущностей,  
- процессов,  
- ролей.

Definition Pack v0.1 уже создан — но необходимо закрепить *архитектурную роль* этого блока.

---

## 2. Решение

**Definition Pack — фундаментальный слой онтологии extended-mind.**  
Он выполняет функции:

1. **Источник терминов** — никакое понятие не используется без определения.  
2. **Гарант смысловой согласованности** — термины не дрейфуют от общения к общению.  
3. **Основной вход для RAG** — машина понимает систему через определения.  
4. **Точка расширения** — новые термины добавляются только сюда.  
5. **База для других документов** — ADR, Patterns, Manifest не вводят терминов ad hoc.

---

## 3. Обоснование

- Без чётких определений невозможна онтология.  
- Extended-mind должен быть машиночитаемым.  
- RAG даёт максимум эффекта при строгих терминах.  
- Автор сможет опираться на консистентную систему.

---

## 4. Последствия

Плюсы:
- система не разваливается на метафоры;  
- понятия не подменяют друг друга;  
- возможен рост сложности;  
- легко обучать новых участников (или других LLM).

Ограничения:
- требует дисциплины — нельзя вводить понятия “на лету”.

---

## 5. Статус

**Принято.**
