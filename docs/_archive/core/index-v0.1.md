# Extended-mind — Core Index v0.1

Этот документ — **карта системы** extended-mind (Мир-3):  
он описывает **архитектурные модули**, их роли и связи.  
Он **не** предназначен для подробного содержания проектов.

---

## 1. Назначение extended-mind (в одном абзаце)

Extended-mind — это смысловое и архитектурное ядро лаборатории utemix-lab:  
структурированный слой знаний (Мир-3), который помогает автору и ИИ-агентам удерживать контекст, принимать решения, превращать сырьё (в т.ч. story-layer) в устойчивые сущности и развивать проекты без расползания в спекуляцию.

---

## 2. Архитектурные уровни (layers)

### 2.1. Knowledge Core (структурный слой)
**Что это:** документы, которые формируют “объективное знание” системы.  
**Что даёт:** определения, решения, паттерны, рамки проектов.  
**Главный принцип:** структура важнее художественного шума.

Состав:
- Definitions / Principles / Manifest
- ADR
- Patterns
- Project Frames
- Concept Maps

---

### 2.2. Story-layer (повествовательный слой)
**Что это:** художественно-рефлексивное сырьё, из которого могут рождаться структурные сущности.  
**Что даёт:** драматургия процесса, материал для авторского нарратива, “пена мысли” как источник кандидатов на тезисы/решения.  
**Главный принцип:** story-layer **не управляет архитектурой напрямую**.

Состав:
- author-line
- machine-log
- story-nodes (как сырьё)

---

### 2.3. Retrieval Layer (RAG v0.1)
**Что это:** механизм поддержки большого контекста через индексирование Knowledge Core.  
**Что даёт:** память, воспроизводимость, “учительский” режим.  
**Главный принцип:** индексируется не “всё”, а **то, на что мы хотим получать ответы**.

Состав:
- ingestion (сканирование → чанкование → эмбеддинги → индекс)
- query-spec / modes / routing / error-handling
- eval (минимальная проверка качества)

---

## 3. Модули ядра (core modules)

### 3.1. Definitions
Роль: “что это?” — канон терминов системы.

### 3.2. ADR
Роль: “почему так?” — конституция архитектурных решений.

### 3.3. Patterns
Роль: “как делать?” — рабочие методы (в т.ч. преобразование story → knowledge).

### 3.4. Project Frames
Роль: “что это за проект?” — каркас проекта без перегруза сырыми деталями.

### 3.5. Concept Maps
Роль: “как всё связано?” — обзорные карты смыслов/проектов.

### 3.6. RAG Core Specs
Роль: “как система себя ведёт?” — режимы retrieval, маршрутизация, обработка ошибок, оценка.

---

## 4. Иерархия авторитета (источник истины)

При конфликтах extended-mind следует этой иерархии:

1) ADR  
2) Project Frames / Ontology (структура мира проектов)  
3) Patterns  
4) Definitions / Principles / Manifest  
5) Story-layer  
6) Свободные рассуждения (reasoning) как последнее средство

---

## 5. Что входит в индекс v0.1 (грубо)

### 5.1. Включаем (по умолчанию)
- Definitions, Principles, Manifest
- ADR
- Patterns
- Project Frames
- Concept Maps
- Core prompts/specs

### 5.2. Исключаем (по умолчанию)
- сырые story-nodes
- author-line сырьё
- машинные логи разработки (machine-log) как поток
- черновики без типа/статуса

---

## 6. Когда архитектура считается “достаточно стабильной”

Система считается стабильной для углубления в контент, если:

1) **Object Types определены** (что такое project, story-node, pattern, adr, …)  
2) **RAG boundary определён** (что индексируем в v0.1 и почему)  
3) **Routing определён** (когда RAG, когда story-layer, когда architect-mode)  
4) Добавление нового проекта требует только:
   - нового `project-frame`,
   - возможных новых tags,
   - (иногда) 1 ADR/Pattern,
   но **не переписывает ядро**.

---

## 7. План развития (минимальный)

- v0.1: один индекс Knowledge Core, ручной ingestion, ручной eval
- v0.2: частичное обновление индекса, отдельный индекс для story-layer (опционально)
- v0.3+: граф/GraphRAG только при явной нужде (триггеры фиксируются ADR)

---

## 8. Триггеры для усложнения (когда думать про граф)

Граф/GraphRAG рассматривается только если:

- количество сущностей и связей стало реально мешать ориентированию,
- RAG часто возвращает “правильные куски, но не те связи”,
- требуется навигация по отношениям (entity→relation→entity) как основной режим.

До этого момента: **RAG + строгая культура документов**.

## 9. Статус

Этот индекс фиксирует архитектуру extended-mind v0.1.  
Он служит “точкой сборки”: после него можно без страха углубляться в содержание проектов.
