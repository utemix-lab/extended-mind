# Machine Log

Этот раздел — фактическая история эксперимента.
Он нужен не для “красоты”, а для ретроспективного восстановления процесса
(контент, отчёты, разборы, рубрики).

## Структура

- `docs/machine-log/kb/` — логи обновлений knowledge base (chunks/embeddings/index)
- `docs/machine-log/dev/` — логи изменения инфраструктуры (PR/релизы/перестройки)

## Принцип

Каждый значимый шаг должен оставить след:
- либо через GitHub history (PR/commit),
- либо через лог-файл, который легко читать человеку.
