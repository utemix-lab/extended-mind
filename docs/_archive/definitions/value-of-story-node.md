# Value of Story-node  
Версия 0.1

---

## 1. Частое заблуждение

Может показаться, что:
- сырые записи,
- эмоциональные наброски,
- “мысленная пена”

— это шум, который нужно удалить, чтобы “не мешал архитектуре”.

Extended-mind принимает прямо противоположную позицию.

---

## 2. Что такое story-node в ценностном смысле

Story-node:
- показывает путь, а не только результат,
- фиксирует реальные сложности мышления,
- сохраняет контекст решения,
- даёт материал для анализа эволюции.

---

## 3. Зачем они нужны extended-mind

Extended-mind использует story-node для:

1. Извлечения тезисов.  
2. Выделения линий.  
3. Распознавания паттернов мышления автора.  
4. Понимания, где возникают тупики и прорывы.

---

## 4. Зачем они нужны автору

Автор:
- может вернуться к моменту, когда идея только рождалась,
- видит, как менялись оценки (“гениально” → “сыро” → “полезно в другом контексте”),
- получает материал для художественной ветви (author-line),
- лучше видит свой собственный стиль мышления.

---

## 5. Зачем они нужны экосистеме

Для utemix-lab:
- это хроника лаборатории,
- материал для обучения других,
- пример честной истории разработки,
- возможная база для будущих публикаций (о мышлении, ИИ, архитектуре).

---

## 6. Принцип неразрушения

Story-node **нельзя**:
- переписывать задним числом,
- “улучшать” их стилистически,
- удалять только потому, что они “неудобные”.

Можно:
- помечать статусом (сырое, неактуальное, пересмотренное),
- делать над ними новые уровни структуризации.

---

## 7. Итог

> Story-node — это не мусор,  
> а дневник работы сознания и extended-mind.  
> Без него система лишается памяти о собственном становлении.
