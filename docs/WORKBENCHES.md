# WORKBENCHES

Типы workbench’ей (без привязки к конкретным персонажам).

## Формат (минимум)

- **ID**
- **Цель**
- **Вход**
- **Выход**
- **Риски**

## Типы (черновик)

### Типы Workbench’ей (v1)
1. **Доменные (контентные)** — живут внутри доменов (Вова, Анимация +)
2. **Метасистемные** — работают над системой (Brand Direction, Circus Technologies)
3. **Коллаборативные / прикладные** — обслуживают систему как продукт (As if)

## Нейминг

### Workbench Naming v1
- Внутрисистемные имена/идентификаторы — **английские** (`workbench_id`, spec ids, file names).
- Нарративные имена и брендинг — **мультиязычные** (RU/EN/JP/гибриды).
- Это не хаос, а отражение цифровой смешанности; канон остаётся на английском.
- Нейминг — часть Brand Direction (Petrova) и отвечает за целостность театра.

### workbench‑signals‑scan
- **Цель:** быстрый обзор новых сигналов
- **Вход:** каталоги/ссылки/теги
- **Выход:** shortlist + story‑node
- **Риски:** медийный захват, каталогизация

### workbench‑tool‑review
- **Цель:** разбор новых инструментов/сервисов
- **Вход:** tool cards, docs, demos
- **Выход:** практика + следы в каталоге
- **Риски:** “второй интернет”, переобъективация

### workbench‑route‑craft
- **Цель:** сбор маршрутов и траекторий
- **Вход:** контекст‑стек, практики, query‑результаты
- **Выход:** saved path + story‑node
- **Риски:** фиксация без смысла
