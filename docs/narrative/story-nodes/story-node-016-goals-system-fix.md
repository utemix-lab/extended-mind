# story-node-016

## Название
Goals framework + System Fix

## Что произошло
Мы ввели систему целей UG/LG/MG/SG и закрепили традицию System Fix как ритуал фиксации изменений.

## Почему это было необходимо
Без карты целей и ритуала фиксации система теряет направление и не сохраняет смысловые сдвиги.

## Принятое решение
Цели формализованы как UG/LG/MG/SG, а System Fix объединяет обзор состояния, чеклисты и публикацию.

## Что изменилось в системе
– добавлены системные документы целей/guardrails/checks  
– введён механизм System Fix  
– закреплены правила сверки изменений с целями

## Открытый вопрос
Нужно ли автоматизировать создание System Fix при крупных PR?

## Что это закладывает
Постоянную навигацию по смыслу, а не по списку задач.
