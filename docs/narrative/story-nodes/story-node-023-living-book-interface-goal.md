# story-node-023-living-book-interface-goal

## Название
Living Book Interface

## Что произошло
Добавлена цель: система воспринимается как живая интерактивная книга‑карта.

## Почему это было необходимо
Чтобы закрепить язык восприятия: книга‑карта вместо документации или каталога.

## Принятое решение
Зафиксировать критерии «живой книги» и связать их с текущими механизмами.

## Что изменилось в системе
- В `SYSTEM_GOALS.md` добавлена цель [UG-5] Living Book Interface.
- В `MECHANISMS_TO_GOALS.md` указаны связи с каталогом, Query Mode, story‑nodes и персонажами.

## Открытый вопрос
Какие UI‑жесты лучше всего показывают «книга живёт», не превращаясь в ленту новостей?

## Что это закладывает
Единый критерий читаемости системы для человека, не знающего архитектуру.
