# story-node-028

## Название
Контекст‑полоса: механика стала видимой

## Что произошло
В интерфейс добавлена context strip с типом ответа и активными контекстами (owner/projection/entity), включая источники и индикатор response type.

## Почему это было необходимо
Механика Query Mode и контекстов была “под капотом”, и без визуального якоря терялся смысл архитектуры.

## Принятое решение
Сделать постоянную визуальную полосу контекста и легенду типов ответа, чтобы интерфейс показывал “как именно он думает”.

## Что изменилось в системе
– появилось отображение активных контекстов в UI  
– добавлен визуальный индикатор типа ответа  
– контексты упорядочены (owner → projection → entity)

## Открытый вопрос
Нужны ли дополнительные уровни объяснения (tooltip/legend) для контекстов?

## Что это закладывает
Стабильную опору для понимания механики и дальнейших UX‑экспериментов.
