# story-node-004-symmetric-highlight

## Название
Симметричная подсветка: виджет ↔ узел

## Что произошло
Реализована полностью симметричная система подсветки: hover на виджет домена и hover на узел графа вызывают идентичную реакцию — узел становится жёлтым, все его рёбра становятся жёлтыми, виджет получает жёлтую рамку и цветную иконку.

## Почему это было необходимо
Пользователь должен интуитивно понимать связь между элементами интерфейса. Асимметричная подсветка создавала когнитивный диссонанс: "почему при наведении на иконку узел не реагирует так же, как при наведении на сам узел?"

## Принятое решение
1. Единая функция `highlightNodeById()` для подсветки узла
2. Единая функция `highlightWidgetById()` для подсветки виджета
3. Оба типа hover (на виджет и на узел) вызывают обе функции
4. Цвет подсветки — `palette.nodeSelected` (жёлтый)

## Что изменилось в системе
– Появилось "золотое правило" симметричной подсветки
– Зафиксирована цветовая семантика: серый → голубой → жёлтый
– Создан документ `INTERACTION_RULES.md` в contracts
– Виджеты используют только внутреннюю подсветку (без box-shadow)

## Открытый вопрос
Как масштабировать это правило на другие типы виджетов (characters, practices)?

## Что это закладывает
"Золотое правило" взаимодействия, которое станет стандартом для всех будущих проекций. Любой новый виджет (character, practice, artifact) автоматически наследует логику симметричной подсветки. UX становится предсказуемым.
