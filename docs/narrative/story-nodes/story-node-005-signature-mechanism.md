# story-node-005-signature-mechanism

## Название

Оригинальность как следы, не как мнение

## Что произошло

Введён механизм signature decisions — маркеры авторского стиля в ADR. Теперь каждое архитектурное решение может быть помечено как формирующее стиль системы: `practice-first`, `perception-ontology-split`, `graph-over-hierarchy` и др.

## Почему это было необходимо

Система формирует уникальный архитектурный стиль, но оригинальность нельзя хранить как декларацию ("мы оригинальны"). Нужен механизм, позволяющий:
- Отслеживать повторяющиеся авторские решения
- Формировать портфолио персонажей через примеры
- Презентовать стиль разным аудиториям

## Принятое решение

ADR расширены секцией `Signature` с маркерами паттернов. Портфолио персонажей = запрос по маркерам, не новые сущности канона.

## Что изменилось в системе

– ADR-001..004 получили signature маркеры
– Создан ADR-005 (механизм signature decisions)
– Создана спецификация SIGNATURE_DECISIONS.md
– Шаблон ADR обновлён

## Открытый вопрос

Как consumer-модуль будет собирать портфолио по маркерам? Достаточно ли grep по файлам, или нужен структурированный индекс?

## Что это закладывает

Основа для автоматического формирования портфолио персонажей. Когда Петрова станет инфлюенсером, её актив = query по маркерам `perception-ontology-split`. Система сможет объяснить свой стиль через примеры, а не декларации.
