# Экосистема utemix-lab — Контекст для LLM

> Этот документ — сжатая выжимка для понимания экосистемы. Загружается в контекст LLM.

## Философия (VISION)

**Backend = экосистема.** Архитектура не скрыта — она видна и является частью интерфейса.

**Graph-first:** Всё представлено как граф связей:
- **SYMBOLIC** — жёсткие связи (теги, ссылки, решения)
- **VECTOR** — мягкие связи (семантическая близость)

**Три слоя интерфейса (3S):**
- **Story** — что происходит (контент, истории, медиа)
- **System** — как устроено (графы, решения, паттерны)
- **Service** — что делать (действия, экспорты)

**Уникальность:** Live-system + Research-through-design. Контент и код развиваются как единое целое.

## Ключевые правила

1. **vovaipetrova-core — read-only канон.** Никаких прямых изменений из других проектов.
2. **Дельты — только предложения.** Автопринятие запрещено.
3. **extended-mind — мета-координатор.** Он не меняет контент, он связывает проекты.

## Типы узлов Universe Graph

### Universe (экосистема)
- `Project` — проект экосистемы
- `Tool` — инструмент (среда со вкладками)
- `Repository` — git-репозиторий

### Architecture (архитектура)
- `Decision` — архитектурное решение (ADR)
- `Pattern` — паттерн / методика
- `Principle` — принцип
- `Contract` — контракт данных

### Ontology (онтология)
- `Concept` — понятие / термин
- `Definition` — определение
- `Layer` — слой системы (3S)

### Actors (роли)
- `Role` — роль в системе
- `Agent` — конкретный агент

### Artifacts (артефакты)
- `Document` — документ
- `Scene` — сцена / вкладка
- `Export` — экспорт данных
- `Episode` — эпизод истории

## Типы связей

### SYMBOLIC (канонические)
- `governs` — A управляет B
- `implements` — A реализует B
- `defines` — A определяет B
- `contains` — A содержит B
- `uses` — A использует B
- `derives_from` — A производная от B
- `feeds` — A питает B

### VECTOR (гипотетические)
- `semantic_near` — A семантически близок к B
- `similar_to` — A похож на B

## Что такое Universe Graph Editor

Визуальный редактор мета-графа экосистемы:
- Редактирование узлов и связей
- Три слоя (3S): Story → System → Service
- RAG-пометки для векторизации
- Структурная валидация
- Два режима чата с LLM:
  - **Meta** — думаем об инструменте
  - **Graph** — думаем графом

## Контекст для режимов LLM

### Meta Mode
LLM — архитектор инструмента. Помогает понять, как должен работать редактор, какие функции нужны, как улучшить UX.

### Graph Mode
LLM — голос экосистемы. Отвечает от лица системы, объясняет связи, находит противоречия, предлагает развитие.
