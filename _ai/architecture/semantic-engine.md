# SEMANTIC ENGINE — Предварительная спецификация

## 1. Назначение
Semantic Engine — это сердце extended-mind.  
Его задача: обеспечивать **процесс мышления**, а не хранение информации.

Он управляет:
- ролями мышления,
- выбором релевантных смыслов,
- связями между lines / ideas / themes,
- диагностикой коллизий,
- подготовкой сущностей к векторизации,
- экспортом смыслов в проекты.

Engine ≠ интеллект.  
Engine = **метаболизм смыслов**.

---

# 2. Основные функции

### 2.1. Retrieval (выбор фрагментов)
Engine выбирает минимальный набор:
- ideas,
- lines,
- themes,
которые нужны модели для текущей задачи.

### 2.2. Role Switching (переключение методов мышления)
Модель может думать как:
- Architect  
- Negative Engineer  
- Fiction Engine  
- Cognitive Mapper  
- Compression Engine  

Engine определяет *правильную последовательность ролей*.

### 2.3. Constraint Awareness
Engine постоянно сверяет мышление с:
- невозможностями,
- ограничениями модели,
- границами архитектуры,
- состоянием проектов.

### 2.4. Semantic Diffusion
Engine распределяет смысл между проектами:
extended-mind → project-A
project-B → extended-mind
extended-mind → project-B

### 2.5. Evolution
Engine позволяет lines:
- ветвиться,
- сжиматься,
- объединяться,
- деградировать,
- становиться themes.

---

# 3. Внутренние модули

## 3.1. Input Interpreter
Понимает, что именно требуется от extended-mind:
- мышление?
- экспорт?
- анализ?
- художественная интерпретация?

## 3.2. Semantic Selector
Находит нужные смысловые узлы:
task → vectors → nearest ideas/lines/themes

## 3.3. Role Orchestrator
Управляет порядком применения ролей.

Пример цепочки:
Negative Engineer → Architect → Fiction Engine → Compressor

## 3.4. Update Logic
Определяет, когда:
- создавать новую idea,
- обновлять line,
- выделять theme,
- архивировать сущность.

## 3.5. Export Compiler
Собирает экспортные сущности:
- narrative
- architecture
- aesthetics
- voice

---

# 4. Принципы работы

### 4.1. Минимальный контекст
Модели не читают extended-mind целиком.

Engine подаёт им «рабочий набор».

### 4.2. Несимметрия ролей
Каждая роль ≠ равнозначна.  
Отрицательный анализ стоит *перед* конструктивным.

### 4.3. Конфликты — топливо
Engine не устраняет конфликты.  
Он использует их как точки роста.

### 4.4. Постепенная векторизация
Сначала — текст.  
Потом — сжатие.  
Потом — embedding.
