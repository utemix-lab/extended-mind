# VECTORIZATION PLAN (Draft)

Цель — создать в будущем **векторную память** extended-mind.

---

# 1. Зачем нужна векторизация

- устойчивое восстановление контекста;  
- поиск смыслов по близости;  
- обнаружение коллизий;  
- идентификация дубликатов;  
- восстановление авторской линии;  
- навигация в огромной смысловой среде.

Текстовый формат → неустойчив.  
Векторный → почти постоянен.

---

# 2. Что будет векторизироваться

## 2.1. Текстовые сущности
- ideas  
- lines  
- themes  
- impossibilities  
- exports

## 2.2. Мультимодальные сущности
- LoRA (через CLIP или адаптацию)  
- визуальные стили  
- голоса персонажей (через аудиомодели)  

## 2.3. Структуры проектов  
- архитектурные слои  
- схемы  
- модели смыслов

---

# 3. Этапы векторизации

### Этап 1 — подготовка (уже идёт)
Стабилизация архитектуры и слоёв.

### Этап 2 — нормализация сущностей
Прогон через Compression Engine.

### Этап 3 — embedding генерация
HF, ColBERT, Faiss, Milvus.

### Этап 4 — semantic memory API
Extended-mind сможет спрашивать:  
**«Покажи похожие линии»**  
**«Где есть аналогичный конфликт»**  
**«Что связано с этой идеей?»**

---

# 4. Принципы векторизации

## 4.1. Ничего не векторизуем, пока не устоялось смыслово  
embedding слабого смысла → мусор.

## 4.2. Вектора — не хранилище, а навигатор  
Текст остаётся главным.

## 4.3. Мультимодальность = обязательна  
Память должна быть многомерной:

текст ↔ визуал ↔ стиль ↔ голос ↔ структура
